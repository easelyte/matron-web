## Appendix: Verified Claims (research pass 2026-07-22)

_Tavily unavailable on this host; c1/c3 verified via WebSearch, c2 from established DOM knowledge._

✓ **Claim (T-3.2 footgun):** A full CSS `font` shorthand value is invalid on the `font-size` property; the browser drops the declaration. Verified: `font-size` accepts only `<length>|<percentage>|<absolute/relative-size>`; a font shorthand requires the `font` property (mandatory size+family, ordering rules), and violating the grammar makes browsers drop the entire declaration. Sources: [MDN `font`](https://developer.mozilla.org/en-US/docs/Web/CSS/font), [W3C validator: "px is not a font-size value"](https://rocketvalidator.com/html-validation/css-font-size-px-is-not-a-font-size-value). ⇒ the plan's `font: var(--cpd-font-body-*)` (whole-style) vs `font-size: var(--cpd-font-size-*)` (size-only) split is correct and load-bearing.

✓ **Claim (T-5.2 pixel-identity gate):** pixelmatch `threshold` ranges 0–1, default 0.1, smaller = more sensitive (larger = more tolerant). Verified across the npm/mapbox reference + Dart/PHP/Python ports. Sources: [pixelmatch npm](https://www.npmjs.com/package/pixelmatch), [mapbox/pixelmatch](https://github.com/mapbox/pixelmatch). ⇒ the plan's "`threshold≈0.1`, not 0%" tolerance is a valid non-zero AA-noise absorber.

✓ **Claim (T-2.5 theme bootstrap):** `MediaQueryList.addEventListener('change', handler)` is the current standard (MediaQueryList extends EventTarget); the older `addListener` is deprecated. Verified: supported in all evergreen browsers (Safari 14+, since 2020). ⇒ the `mq.addEventListener("change", applyTheme)` bootstrap is correct; no `addListener` fallback needed for the target (Tailscale-served desktop/modern) browsers.
